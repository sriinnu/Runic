## Expected Behavior

### TypeScript Compilation
- Should complete with no errors
- Should produce files in `dist/` directory

### Server Startup
- Server will start and print: "Runic [Server Name] v2.0.0 running on stdio"
- Server will wait for MCP protocol messages via stdin (this is normal)
- Press Ctrl+C to exit

### Build Artifacts

Each server should have these files in `dist/`:

**All Servers:**
- `index.js` - Main server file with shebang line
- `index.d.ts` - Type definitions
- `tools.js` - Tool implementations
- `tools.d.ts` - Tool type definitions
- `schemas.js` - Zod schema definitions
- `schemas.d.ts` - Schema type definitions

**Persistence Server Only:**
- `database.js` - SQLite database wrapper
- `database.d.ts` - Database type definitions

## Troubleshooting

### TypeScript Errors

If you see TypeScript compilation errors:
1. Check that all source files exist in `src/`
2. Run `npm install` to ensure all dependencies are present
3. Check `tsconfig.json` for correct settings

### Build Failures

If build fails:
1. Delete `dist/` directory and rebuild
2. Run `npm install` to refresh dependencies
3. Check for syntax errors in source files

### Runtime Errors

If server crashes on startup:
1. Check that all dependencies are installed
2. Verify Node.js version is compatible (requires Node.js 16+)
3. Look for missing import statements

### Import Errors

All imports use `.js` extensions even for TypeScript files:
- ✅ `import { X } from "./file.js"`
- ❌ `import { X } from "./file"`

This is required for ES modules in Node.js.
