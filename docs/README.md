# Runic Docs

This folder is for durable public documentation only.

Keep:
- Product architecture and provider behavior docs.
- CLI, release, packaging, status, UI, widget, and design-system docs.
- GitHub Pages assets used by `docs/index.html`.

Do not commit:
- Generated audits, implementation summaries, quick references, or progress reports.
- One-off research, exploratory mockups, old screenshots, or local test harnesses.
- Auth dumps, dashboard captures, local paths, account emails, org IDs, or provider secrets.

Local-only notes belong in ignored paths such as `docs/research/`, `docs/workflows/`, `docs/design/mockups/`, or a private scratch directory outside the repo.
